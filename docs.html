Documentation
There are three different ways to use dat: on the CLI (command line), using HTTP with the dat REST API, or from Node.js with the dat JavaScript API.
All of the documentation linked to below is hosted on the dat GitHub repository. If you find bugs or have questions please let us know by opening an issue or sending a pull request.
The best place to start is the getting started guide. We also have detailed API documentation:

FAQ

What would I use it for? Is it a replacement for mongodb or sql or csv files?
Dat is not a replacement for mongodb/sql/csv. Dat is an interface for these different storage formats and database types so that the consumer doesn't have to care about in which particular protocol the data is stored.
If it is not a replacement for things like mongodb/sql/csv flies, how would I use it along side of existing workflows where I am using mongo/sql/csv?
Right now we are working on two different ways to sync or connect your existing workflows.
1. Datscript: this is a simple module that lets you set up data pipelines. This is exceptionally great for datasets that live in a very hard to reach place, like excel files on FTP servers, where a custom set of update scripts must be run to get the data in the proper ndjson format, streamable to dat.
2. Real-time sync: We are building a sync function that will pull data from a connected datastore to your configured dat. The interface is still in the prototype stages.
Is dat designed for production data pipelines or more for sharing data in interactive/exploratory workflows?
Right now, our target users are researchers, analysts, and data enthusiasts who need a way to share data. I can imagine some lovely production data pipeline scenarios, though, when the time comes. You can see some of our use cases here.
If I have multiple different tables that I want to store, would I create a separate dat repo for each? Can a single dat repo store multiple tables?
Each dat repo is only one table. Features we are working on will be put on the beta roadmap. You should request a feature!
How would someone use data in dat. Just doing "dat cat"? Are there Python libraries that tie into pandas or another Python library?
We're building that right now. Dat has a REST as well as a native interface, and we're planning on supporting both really soon.
