Versioned data, collaborated

Dat is a version-controlled, distributed data tool designed to improve collaboration between data people and data systems.

Version by row or by file

Add data into dat from newline-delimited json, csv, or binary format. You can version the data by row or by file. Learn more >>

$ dat parse values_per_city.csv
Added 302,143 keys (32.03 Mb, 4.4 Mb/s).
Checked out to version bad31c.

$ dat write model_data.tar.gz
Added model_data.tar.gz (153.03 Mb, 15.3 Mb/s).
Checked out to version cef322.
SSH and HTTP endpoints

Expose your dat through the filesystem with SSH or HTTP. This means access control comes free and easy. Learn more >>

$ dat clone ssh://uni.edu/maxogden/cities
Organize by dataset

Add dat.json files to your git repository to keep track of versioned data with versioned code. Datasets are stored in a named, hierarchical folder structure. Learn more >>

$ ls -a
dat.json
.git/
analysis.py

$ dat pull

Clone from remote has completed.
Checked out to c2342de3f

Stream anything anywhere

Data is streamed into the Dat database, one row or file at a time. That means that your computer does not have to hold the entire dataset in memory, and you can stop a pull or clone at any time. Learn more >>

$ sql2csv "SELECT * from us_cities" | dat parse
Added 302,143 keys (32.03 Mb, 4.4 Mb/s).
Data added successfully.
